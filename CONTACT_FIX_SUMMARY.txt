⚙️  REQUIREMENTS FOR FULL FUNCTIONALITY
─────────────────────────────────────────────────────────────────────────────

For email sending to work, you need to add these to Vercel environment variables:

Backend Requirements:
  • SUPABASE_URL (from Supabase project settings)
  • SUPABASE_KEY (from Supabase project settings)
  • EMAIL_HOST=smtp.gmail.com
  • EMAIL_PORT=587
  • EMAIL_SECURE=false
  • EMAIL_USER=your_email@gmail.com
  • EMAIL_PASS=your_app_password (generate from Gmail)
  • EMAIL_FROM=noreply@example.com
  • EMAIL_TO=your_email@gmail.com

Steps:
  1. Go to Vercel Dashboard
  2. Select your project
  3. Settings → Environment Variables
  4. Add each variable above
  5. Redeploy the project

📧 DATABASE SCHEMA (contact_messages)
─────────────────────────────────────────────────────────────────────────────

CREATE TABLE public.contact_messages (
  id serial NOT NULL PRIMARY KEY,
  name varchar(150) NOT NULL,
  email varchar(150) NOT NULL,
  subject varchar(255),
  message text NOT NULL,
  ip_address varchar(45),
  status varchar(20) DEFAULT 'unread',
  created_at timestamp DEFAULT CURRENT_TIMESTAMP,
  read_at timestamp
);

🐛 TROUBLESHOOTING
─────────────────────────────────────────────────────────────────────────────

Error: "Failed to send message. Please try emailing directly."
  → Check browser console (F12) for detailed error
  → Verify all form fields are correctly filled
  → Check backend logs on Vercel
  → Ensure Supabase credentials are set

Error: "Email is not valid" (red message)
  → Email format is incorrect
  → Use format: yourname@example.com
  → Check for extra spaces

Error: Message not saving to database
  → Verify SUPABASE_URL and SUPABASE_KEY in Vercel
  → Check Supabase table permissions
  → Redeploy the project

Rate limit error
  → Wait 15 minutes before sending another message
  → Each IP can send max 5 messages per 15 minutes
