🔄 KEY TECHNICAL CHANGES
═══════════════════════════════════════════════════════════════════════════════

OLD WAY (MySQL):
  const [rows] = await pool.query('SELECT * FROM profile LIMIT 1');

NEW WAY (Supabase):
  const { data, error } = await supabase
    .from('profile')
    .select('*')
    .limit(1)
    .single();

OLD ENVIRONMENT VARIABLES:
  DB_HOST=localhost
  DB_PORT=3306
  DB_USER=root
  DB_PASSWORD=
  DB_NAME=cv

NEW ENVIRONMENT VARIABLES:
  SUPABASE_URL=https://your-project.supabase.co
  SUPABASE_KEY=your-anon-key

ALL QUERY PATTERNS CONVERTED:
  • SELECT queries → .select()
  • WHERE clauses → .eq(), .gte(), .lte(), etc.
  • ORDER BY → .order()
  • INSERT → .insert()
  • Complex JOINs → Nested Supabase selects
  • JSON handling → PostgreSQL JSONB (better performance)


🚀 HOW TO GET STARTED
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Get Supabase Credentials (2 minutes)
  1. Go to https://app.supabase.com
  2. Find your project settings
  3. Copy: Project URL (from Settings → General)
  4. Copy: Anon Key (from Settings → API)

STEP 2: Create Database Schema (1 minute)
  1. In Supabase, go to SQL Editor
  2. Create new query
  3. Open file: database/supabase-schema.sql
  4. Copy all SQL code
  5. Paste into Supabase SQL Editor
  6. Click Run

STEP 3: Setup Backend (2 minutes)
  1. Open terminal in backend/ folder
  2. Run: npm install
  3. Create .env file with:
     SUPABASE_URL=https://your-url.supabase.co
     SUPABASE_KEY=your-anon-key
     PORT=5000
     NODE_ENV=development
     CORS_ORIGIN=http://localhost:3000
     (+ email settings from .env.example)

STEP 4: Start Backend (1 minute)
  1. Run: npm run dev
  2. You should see:
     ✅ Supabase PostgreSQL connected
     🚀 Backend API running on http://localhost:5000

STEP 5: Test
  1. Browser: http://localhost:5000/health
  2. Terminal: curl http://localhost:5000/api/profile

🆘 NEED HELP?
═══════════════════════════════════════════════════════════════════════════════

Common Issues & Quick Fixes:

  ❌ "Missing Supabase credentials"
     → Check .env file exists with SUPABASE_URL and SUPABASE_KEY

  ❌ "Supabase connection failed"
     → Verify credentials from Supabase dashboard
     → Check internet connectivity
     → Ensure schema was imported

  ❌ "Empty results from API"
     → Check data exists in Supabase Studio
     → Verify table names (case-sensitive in PostgreSQL)

  ❌ "CORS error in frontend"
     → Update CORS_ORIGIN in .env to match your frontend URL

See MIGRATION_GUIDE.md for more troubleshooting tips.
